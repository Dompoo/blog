### 1. 스프링 컨테이너 시작
- 시스템 환경변수와 `application.yml`같은 설정 정보를 읽어들인다.
- 이를 통해 환경 객체를 먼저 만든다.
### 2. 빈 설정 정보 스캔
- XML 구성 / 자바 구성 / 애노테이션 구성 정보들을 모두 읽어서 [[BeanDefinition]]을 만든다.
### 3. BeanFactoryPostProcessor 실행
- 실제 빈이 생성되기 직전, [[BeanDefinition]] 자체를 수정한다.
- 예를 들어, `${database.url}`같은 플레이스 홀더를 실제 값으로 치환해준다.
### 4. 빈 인스턴스화
- 실제 빈 객체를 힙 메모리에 생성한다.
- [[BeanDefinition]]의 의존 관계에 따라 의존이 없는 객체부터 만들어간다.
- 이때 [[생성자 의존관계 주입 방법|생성자 주입]]은 자연스럽게 수행된다.
### 5. BeanPostProcessor - Before Initialization
- [[빈 생명주기 콜백]]의 `@PostConstruct` 가 되기 전에, 처리 작업이 일어난다.
- 초기화를 위한 준비물을 챙기는 역할이다.
